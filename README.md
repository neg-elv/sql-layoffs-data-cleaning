# SQL Data Cleaning Project – Layoffs Dataset

## Overview
This project focuses on cleaning and preparing a real-world layoffs dataset using SQL in MySQL Workbench.

The main goal was to transform raw and inconsistent data into a clean and analysis-ready dataset by performing several data cleaning operations commonly used in real data analytics workflows.

---

## Objectives
The project includes:

- Removing duplicate records
- Standardizing inconsistent values
- Cleaning NULL and blank values
- Formatting date columns correctly
- Improving overall data quality
- Preparing the dataset for exploratory data analysis (EDA)

## Dataset Information
The dataset contains layoff information from companies around the world, including:

- Company names
- Industry
- Country and location
- Total employees laid off
- Percentage laid off
- Dates
- Funding information

---

## Project Workflow

### 1. Creating a Staging Table
A staging table was created to safely clean and transform the raw dataset without affecting the original data.

### 2. Removing Duplicate Records
Duplicate rows were identified and removed using SQL window functions and row numbering techniques.

### 3. Standardizing Data
Inconsistent values in columns such as:
- industry
- country
- company names

were standardized to ensure consistency across the dataset.

### 4. Handling NULL and Blank Values
Missing and blank values were cleaned or updated where possible using SQL joins and conditional logic.

### 5. Formatting Dates
Date columns were converted into proper SQL date formats for easier analysis and querying.

### 6. Final Data Preparation
The cleaned dataset was prepared for future exploratory data analysis and visualization projects.
